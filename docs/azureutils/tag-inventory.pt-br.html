Inventário de tags

Exporte um inventário de recursos + tags do Azure para uma planilha Excel, edite-a e aplique as mudanças de volta no Azure. Dois cmdlets formam um round-trip seguro: Export-AzureUtilsTagInventory e Set-AzureUtilsTagInventory.

Export-AzureUtilsTagInventory

Exporta um inventário de recursos + tags do Azure para uma planilha Excel via Azure Resource Graph. O cmdlet não emite objetos — escreve o .xlsx e imprime um relatório no console. O escopo pode ser restringido por management group(s), subscription(s), resource group(s) e um match case-insensitive -NameContains.

Parâmetros

ParâmetroMúltiplo?Finalidade
-ManagementGroupIdsim (string[])Um ou mais management groups.
-SubscriptionIdsim (string[])Uma ou mais subscriptions (padrão: todas habilitadas no contexto).
-ResourceGroupNamesim (string[])Filtra por nome(s) de resource group.
-NameContainsnão (termo único)Mantém recursos cujo nome contém este texto.
-FilterTagssim (string[])Exporta apenas estas chaves de tag, nesta ordem (padrão: todas as tags encontradas).
-IncludeSubscription—Adiciona também as subscriptions como linhas (com suas próprias tags).
-IncludeResourceGroup—Adiciona também os resource groups como linhas (com suas próprias tags).
-IncludeTagUnsupported—Mantém recursos cujo tipo não suporta tags e adiciona uma coluna colorida Tag Support.
-OutputPathobrigatórioArquivo .xlsx de destino.
-TableStyle—Estilo da tabela Excel (padrão neutro Light1).
-Quiet—Suprime as linhas de log por recurso (mostra uma barra de progresso).
⚠
-ManagementGroupId e -SubscriptionId são mutuamente exclusivos (parameter sets separados). Requer o módulo ImportExcel.

Exemplos

pwsh — Export-AzureUtilsTagInventory
Connect-AzAccount

# 1) Tudo no contexto atual, todas as tags como colunas
Export-AzureUtilsTagInventory -OutputPath '.\inventory.xlsx'

# 2) Um management group, apenas duas tags (colunas nesta ordem)
Export-AzureUtilsTagInventory -ManagementGroupId 'PLAT' `
    -FilterTags 'costCenter', 'environment' `
    -OutputPath 'C:\Temp\report.xlsx'

# 3) Vários management groups de uma vez
Export-AzureUtilsTagInventory -ManagementGroupId 'PLAT', 'SANDBOX' -OutputPath '.\all.xlsx'

# 4) Escopo restrito + suprime o log por recurso (barra de progresso)
Export-AzureUtilsTagInventory -SubscriptionId $sub1, $sub2 `
    -ResourceGroupName 'rg-prod' -NameContains 'sql' `
    -OutputPath '.\sql-prod.xlsx' -Quiet

# 5) Inventaria também as subscriptions e os resource groups
Export-AzureUtilsTagInventory -ManagementGroupId 'PLAT' `
    -IncludeSubscription -IncludeResourceGroup `
    -OutputPath '.\full.xlsx'

# 6) Mantém tipos sem suporte a tags e os sinaliza
Export-AzureUtilsTagInventory -IncludeTagUnsupported -OutputPath '.\inventory.xlsx'
ℹ
Por padrão apenas tags no nível de recurso são inventariadas. -IncludeSubscription e -IncludeResourceGroup adicionam as subscriptions / resource groups como linhas extras (de resourcecontainers) para capturar tags aplicadas nesses escopos. Recursos cujo tipo não suporta tags (ex.: networkWatchers, recursos clássicos) são omitidos por padrão — -IncludeTagUnsupported os mantém e adiciona uma coluna Tag Support (Supported verde / Not supported vermelho), resolvida via API de Resource Providers do ARM.

Layout do Excel: colunas fixas resourceId, Resource Name, Sub Name, Resource Group Name, Resource Type, Region, seguidas de uma coluna TAG_<nome> por tag. Sem -FilterTags, cada chave de tag encontrada vira uma coluna (união ordenada); com -FilterTags, só as chaves listadas aparecem, na ordem dada (em branco onde o recurso não tem aquela tag). Mude a aparência com -TableStyle (qualquer estilo do ImportExcel).

Set-AzureUtilsTagInventory

Lê um .xlsx de inventário (o produzido por Export-AzureUtilsTagInventory, opcionalmente editado) e aplica os valores TAG_<nome> de volta em cada recurso identificado pelo resourceId. A operação é um merge (Update-AzTag -Operation Merge):

  • tags presentes no recurso mas ausentes do arquivo são mantidas (nunca removidas);
  • uma célula TAG_<nome> deixada vazia é ignorada (nunca criada ou alterada);
  • uma coluna TAG_<nome> adicionada manualmente (com valor) cria essa tag.

Este cmdlet altera recursos do Azure, então suporta -WhatIf e -Confirm. As linhas são agrupadas por subscription (o contexto é trocado por grupo). Requer Az.Resources.

ParâmetroFinalidade
-InputPath (obrigatório, pos. 0)Caminho do .xlsx a ler (alias -Path).
-WorksheetNamePlanilha a ler (padrão TagInventory).
-QuietSuprime as linhas de log por recurso (erros ainda aparecem).
pwsh — Set-AzureUtilsTagInventory
# Pré-visualiza as mudanças sem alterar nada
Set-AzureUtilsTagInventory -InputPath 'C:\Temp\report.xlsx' -WhatIf

# Aplica (edite as células TAG_* / adicione colunas TAG_<nova> antes)
Set-AzureUtilsTagInventory 'C:\Temp\report.xlsx'

Round-trip

↻
Export-AzureUtilsTagInventory → edite a planilha (mude valores de tags, adicione colunas TAG_<nova>) → Set-AzureUtilsTagInventory para aplicar. A semântica de merge torna isso seguro: células em branco e tags não tocadas nunca são destruídas.
