AzureUtils

Practical utility cmdlets for Azure administration, governance, inventory, troubleshooting and operational automation. AzureUtils does not replace the Az modules — it encapsulates real day-to-day pain (inventory at scale, cross-cutting diagnostics, governance checks, safe fixes) behind simple commands with normalized, automation-ready output.

What it is

AzureUtils is a focused toolbox for people who operate Azure every day. Where the official Az modules give you the raw building blocks, AzureUtils gives you the outcomes — a tag inventory you can edit in Excel and push back, a list of resources that look orphaned, the resources exposed to the public internet — each behind a single command with clean, pipeline-friendly output.

Conventions

  • PowerShell 7+ only (CompatiblePSEditions = Core).
  • Get-* commands return objects; Export-* commands write files and print a console report.
  • Read commands use Get-* / Find-*; mutating commands support -WhatIf / -Confirm.
  • All command output and messages are in en-US.

Requirements

All three modules below are declared as RequiredModules, so Install-Module pulls them in automatically.

ModuleRequiredPurpose
Az.AccountsYesResolves the Azure context (Connect-AzAccount).
Az.ResourceGraphYesPowers the inventory query (multi-scope, auto-paginated).
ImportExcelYesUsed by Export-AzureUtilsTagInventory to write the .xlsx.
ℹ
Set-AzureUtilsTagInventory additionally requires Az.Resources to apply tags back to Azure.

Install

pwsh
# Stable — also installs Az.Accounts, Az.ResourceGraph, ImportExcel
Install-Module AzureUtils -Scope CurrentUser

# Pre-release
Install-Module AzureUtils -AllowPrerelease -Scope CurrentUser
